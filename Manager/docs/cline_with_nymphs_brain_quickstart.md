# Cline With Nymphs-Brain Quickstart

This guide connects VS Code Cline to the optional `Nymphs-Brain` local LLM and MCP tools installed by NymphsCore Manager.

Use this after:

- NymphsCore Manager finished installing successfully
- the optional `Nymphs-Brain` module was selected
- the fresh install is running inside the managed `NymphsCore` WSL distro

## What You Will End Up With

Cline can use:

- the local LM Studio-backed coding model through `http://localhost:1234/v1`
- the local Brain MCP gateway through `http://localhost:8100`

That gives you:

- local chat/coding with the Brain model
- filesystem tools
- memory tools
- web-forager tools

## Before You Start

From Windows PowerShell, make sure the Brain services are up:

```powershell
wsl -d NymphsCore --user nymph -- bash -lc "/home/nymph/Nymphs-Brain/bin/lms-start"
wsl -d NymphsCore --user nymph -- bash -lc "/home/nymph/Nymphs-Brain/bin/mcp-start"
```

Optional check:

```powershell
wsl -d NymphsCore --user nymph -- bash -lc "/home/nymph/Nymphs-Brain/bin/brain-status"
```

Expected healthy output looks like:

```text
Brain install: installed
LLM server: running
Model loaded: qwen/qwen2.5-coder-14b, ...
MCP proxy: running
Open WebUI: running|stopped
```

`Open WebUI` does not need to be running for Cline MCP to work.

## Part 1: Connect Cline To The Local Model

In Cline settings:

1. open `API Configuration`
2. set `API Provider` to `OpenAI Compatible`
3. fill in:

```text
Base URL: http://localhost:1234/v1
OpenAI Compatible API Key: lm-studio
Model ID: qwen/qwen2.5-coder-14b
```

Then click `Done`.

Important:

- do not use the built-in `LM Studio` provider if you want MCP tools too
- use `OpenAI Compatible` instead

Quick test in a new Cline chat:

```text
Reply with exactly these 5 words: local model test successful
```

Expected response:

```text
local model test successful
```

## Part 2: Add MCP Servers In Cline

In the Cline sidebar:

1. open `MCP Servers`
2. open the `Remote Servers` tab
3. add these one at a time with transport `Streamable HTTP`

### Filesystem

```text
Server Name: filesystem_mcp
Server URL: http://localhost:8100/servers/filesystem/mcp
Transport Type: Streamable HTTP
```

### Memory

```text
Server Name: memory_mcp
Server URL: http://localhost:8100/servers/memory/mcp
Transport Type: Streamable HTTP
```

### Web Forager

```text
Server Name: web_forager_mcp
Server URL: http://localhost:8100/servers/web-forager/mcp
Transport Type: Streamable HTTP
```

After adding them, open the `Configure` tab. You should see:

- `filesystem_mcp`
- `memory_mcp`
- `web_forager_mcp`

Each one should show a green status dot when healthy.

## What Each Tool Does

`filesystem_mcp`

- lets Cline read and work with files inside the allowed WSL paths

`memory_mcp`

- gives Cline a lightweight memory tool server

`web_forager_mcp`

- gives Cline web/fetch style MCP tools from the Brain gateway

## Common Gotchas

### Cline says GPT-4 or acts like the old provider is still active

Usually this means the old chat kept the previous provider context.

Fix:

1. click `Done` after saving settings
2. start a brand new Cline chat
3. test again

If needed, reload VS Code:

```text
Developer: Reload Window
```

### The built-in LM Studio provider says it does not support MCP

That is expected.

Use:

```text
API Provider: OpenAI Compatible
```

Not:

```text
API Provider: LM Studio
```

### A server name is rejected as invalid

Use simple underscore names such as:

```text
filesystem_mcp
memory_mcp
web_forager_mcp
```

### MCP servers do not connect

Check the gateway first:

```powershell
wsl -d NymphsCore --user nymph -- bash -lc "/home/nymph/Nymphs-Brain/bin/mcp-start"
wsl -d NymphsCore --user nymph -- bash -lc "/home/nymph/Nymphs-Brain/bin/brain-status"
```

### Cline warns about checkpoints in the home directory

That warning is about your VS Code workspace location, not the Brain connection itself.

The local model and MCP tools can still work even if that warning is present.

## Files Generated By The Installer

The Brain install also writes reference files here:

```text
/home/nymph/Nymphs-Brain/mcp/config/cline-mcp-settings.json
/home/nymph/Nymphs-Brain/mcp/config/mcp-proxy-servers.json
```

These are useful if you want to compare Cline's saved config with the installed Brain defaults.

## Summary

If everything is healthy:

- Cline uses `OpenAI Compatible`
- Base URL is `http://localhost:1234/v1`
- model is `qwen/qwen2.5-coder-14b`
- MCP gateway is `http://localhost:8100`
- three remote MCP servers are added in Cline:
  - `filesystem_mcp`
  - `memory_mcp`
  - `web_forager_mcp`
