`Nymphs3D2` installer path and re-testing model pulling on a clean install.

- a fresh installer-created `Nymphs3D2` distro now opens as:
  - `nymphs3d`

## Main Repo Changes In This Session

### Fresh Install Runtime Layout Fix

Files:

- `scripts/import_base_distro.ps1`
- `scripts/finalize_imported_distro.sh`
- `scripts/prepare_fresh_builder_distro.sh`

What changed:

- after import, the distro user is now configured and the distro is restarted
  so the default user setting actually takes effect
- fresh imports now also normalize `/etc/profile.d/nymphs3d.sh` so runtime
  paths point to the user home instead of the old `/opt/nymphs3d/runtime/...`
  location
- the builder-prep path now writes the same home-based runtime shell exports,
  so future exported base distros do not carry the stale `/opt` runtime root

- the `More Info` section uses simple full-width buttons

## Launcher State

The launcher source is now tracked in the main repo here:

- `launcher/build_launcher_windows.ps1`
- `launcher/frontend/hunyuan_launcher.py`

Key local launcher changes made during this session:

- backend log pane now behaves like a real backend terminal by default
- backend processes are started through a pseudo-terminal wrapper using:
  - `script -qefc ... /dev/null`
- this makes Hugging Face / tqdm download progress behave much more like the
  plain WSL terminal path
- the launcher now exposed actual first-run progress such as:
  - `Downloading (incomplete total...): 1% 67.0M/9.86G ...`

Open launcher work still remaining:

- the generic `[progress] Still working...` line is now less important and may
  be worth toning down later
- first-run download feedback is far better than before, but backend-side
  `snapshot_download(...)` behavior is still noisy and not ideal
