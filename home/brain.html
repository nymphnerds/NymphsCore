32K context

Adds roughly 2-4GB VRAM overhead through KV cache growth.

64K context

Adds roughly 4-8GB VRAM overhead.

128K context

Adds roughly 8-16GB VRAM overhead, depending on model architecture.

Qwen 3.5 / 3.6 note

Gated Deltanet reduces KV cache growth in many layers, so long contexts are often more achievable than the raw table suggests.

MoE models

Still need full model VRAM to load, but only part of the network activates per token during inference.

Rough VRAM formula

(params x 0.5 for Q4 / x 0.7 for Q6) + 2GB overhead + context overhead

// Quick start

Start fast.
Stay oriented.

Once you know what your hardware can carry, these are the commands you will reach for most often when bringing the local stack up, shutting it down, or checking what is already running.

01
Start everything

Boot LM Studio, the MCP gateway, and the chat UI in one go.

./bin/brain-start
02
Stop everything

Cleanly shut down the full stack when you are done working.

./bin/brain-stop
03
Check status

See what is running before you debug or start services again.

./bin/brain-status
Default setup snapshot: LM Studio runs on model qwen/qwen3.5-27b with a 100,000 token context window, MCP listens on 8100, and Open WebUI is exposed on 8081.
// Commands

All commands.
Broken into real groups.

Everything is grouped by the part of the stack it controls, so startup, model serving, gateway tools, and chat access are easy to find at a glance.

Main Commands

Use these first when you want to start, stop, inspect, or prime the whole environment.

CommandWhat It Does
brain-startStarts all services: LM Studio, MCP Gateway, and Open WebUI.
brain-stopStops all services.
brain-statusShows which services are running.
brain-envSets up environment variables for the system.

LM Studio Commands

This is the local model server layer, including model selection and launch control.

CommandWhat It Does
lms-modelInteractive menu to manage models, including download, selection, and removal.
lms-startStarts the AI model server and loads a model.
lms-stopStops the AI model server.

Default settings: model qwen/qwen3.5-27b, context length 100,000, port 1234.

MCP Gateway Commands

This layer exposes the tool server with filesystem, memory, web-forager, and context7.

CommandWhat It Does
mcp-startStarts the MCP gateway with 4 tools: filesystem, memory, web-forager, and context7.
mcp-statusShows MCP status and URLs.
mcp-stopStops the MCP gateway.

Default settings: port 8100, URL http://localhost:8100.

Open WebUI Commands

This is the local chat layer for talking to your running stack once the services are up.

CommandWhat It Does
open-webui-startStarts the chat interface.
open-webui-stopStops the chat interface.

Default settings: port 8081, URL http://localhost:8081.

// Tips

Small habits.
Less friction.

A few simple habits make the local setup easier to manage day to day, especially when you are bouncing between startup, testing, and shutdown.

  • Run brain-start to get everything going at once.
  • Use brain-status to see what is running before troubleshooting.
  • Run brain-stop when you are done.
  • All commands live in the bin/ folder of Nymphs Brain.
