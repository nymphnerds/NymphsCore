// The pipeline

Every step.
One workflow.

The sequence matters: image generation, image editing, and part extraction happen first in Nymphs Image. Then those images move into shape generation and retexturing.

01
🖼️

Local text-to-image with reusable subject and style presets

✏️
Edit the image
Use a guide image to push a reference closer to the design you actually want
03
🧩
Extract parts
Plan and extract modular parts inside Nymphs Image before the shape pass starts
04
📦

Turn the chosen image or extracted part references into textured 3D drafts with TRELLIS.2
05
🎨
Retexture
Second-pass texturing when the first draft needs a different surface treatment
// NymphsCore Manager

Setup hell,
solved.

Most local AI falls apart before it starts. The NymphsCore Manager handles the install and repair work so artists can focus on the pipeline instead of the plumbing.

  • ✓WSL setup and dedicated distro creation
  • ✓CUDA and Python environment prep
  • ✓Model prefetch, runtime checks, and smoke tests
  • ✓Repair flows and refresh when the local stack needs help
  • ✓One coherent path instead of building your own backend stack

// Who it's for

Honest about
the fit.

NymphsCore is stronger when it is specific. That focus is part of why the product story works.

Strong fit
  • Indie game artists in Blender
  • Solo devs needing faster concept-to-asset
  • Small teams that value local ownership
  • Windows + NVIDIA GPU setups
  • Artists tired of building their own pipeline
Weaker fit
  • Mac-first workflows
  • Non-NVIDIA GPU users
  • Teams wanting pure cloud pipelines
  • Expecting final production topology
