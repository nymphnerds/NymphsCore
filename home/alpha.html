// What to include

Small details make bugs fixable.

Good alpha reports do not need to be polished. Include the exact step, what happened, and any screenshots or logs you can paste.

  • Windows version, GPU, Blender version, and NymphsCore build if known.
  • The exact button, panel, command, or step that failed.
  • What you expected versus what actually happened.
  • Logs when install or runtime fails.

Submit sends the report directly to the NymphsCore alpha feedback inbox. No GitHub account required.
