These are the practical defaults and habits we wanted carried into the calmer NymphsCore front end. This page is still a simplification, not a 1:1 clone of every upstream control.
  • Learning rate: 1e-4 is the main safe starting point shown in the walkthrough.
  • Optimizer: AdamW8Bit with weight_decay 1e-4 is the default direction.
  • Timesteps: use weighted sampling and start from a more balanced style/content mix before pushing harder.
  • Loss: MSE stays on. EMA stays off for this path.
  • Caching: cache_text_embeddings on, unload text encoder off.
  • Preservation toggles: differential output preservation and blank-prompt preservation were not part of the recommended baseline.
  • For style LoRAs, caption the scene normally. Let the images teach the rendering style instead of stuffing captions with repeated style labels.
  • Triggers are optional. Some LoRAs want one, some do not. Test with and without instead of assuming trigger-word theory is law.
  • Start balanced, then push style later. Use the first run to prove the signal before you jump to stronger high-noise behavior.
  • The first run is for signal, not perfection. If the LoRA clearly bends output in the right direction, that is enough information to tune the next pass.
