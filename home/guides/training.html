Guide 05

Train a Z-Image LoRA.Start safe, then push style on purpose.

This is the beginner guide for the new trainer flow. It is written to be practical, not academic: what captions actually do, how to set up a first test on a 4080 Super, and how to tell whether a run is weak, healthy, or overcooked.

// Quick Read

The short version.

The trainer uses the same Turbo training adapter approach we discussed from Ostris, and the Manager now treats captions as user-owned training data. That means the app helps you open and manage the captions file, but it does not silently write the meaning of your dataset for you.

  • Character / Object is for identity, objects, or repeatable subjects.
  • Stylized Look is for broader rendering shifts like kids drawings or strong illustration language.
  • Quick Test is your system check and first sanity pass, not the final quality run.
// Why Captions Matter

The images teach the model what things look like. The captions teach it what words should call those things back up later. If the caption is vague, noisy, or wrong, the LoRA can still learn something, but it will be much harder to prompt cleanly.

  • The caption is not busywork. It is part of the training pair.
  • You do not need perfect prose. You do need clear intent.
  • For style work, describe the subject normally and let the LoRA learn the rendering style.
// Captions

What captions are actually doing.

A caption is the text side of the training example. When the trainer sees an image of a bear in a childlike crayon drawing and the caption says a bear building a log cabin in the snow, it learns how that subject and wording connect. Later, when you prompt for a bear building a cabin, the LoRA has something meaningful to grab onto.

Rule 01

Describe the subject, action, and key read.

Write what is in the image in the way you would want to prompt it later. This is usually more useful than stuffing in every technical detail you can think of.

  • Good: a child holding a red kite in a windy field
  • Good: a rabbit reading a book at a wooden desk
  • Weaker: nice art style cute image
Rule 02

Do not let the app invent the captions for you.

That is why the Manager now opens metadata.csv for you instead of auto-writing caption text. If captions are important, they need to stay in your hands.

  • The Manager will keep the image filenames in sync.
  • It will preserve captions you already wrote.
  • It will block training if captions are still blank.
Rule 03

For stylized training, caption the content normally.

This is the big idea from the kids-drawing style workflow. If every caption just repeats the style name, the LoRA can become less useful. For strong styles, it is often better to caption the content cleanly and let the LoRA absorb the visual treatment from the images themselves.

If you are unsure what to write, aim for one clear sentence per image. Subject first, then action, then the most important visible qualifiers.
// Training Focus

Character / Object

Use this when you want a person, creature, object, or repeatable identity to come through more strongly. This mode keeps a full-range training emphasis and is the safer choice when the goal is consistency of subject.

// Training Focus

Stylized Look

Use this when the point is a broad rendering language, like kids drawings, heavy illustration, or a very particular visual voice. This mode now pushes training toward earlier, higher-noise timesteps so the style can influence more than just surface texture.

// First Test

Your first run on a 4080 Super.

The goal of your first training run is not perfection. It is to confirm the pipeline works, that your captions make sense, and that the LoRA is moving in the right direction before you invest in a longer run.

Step 01

Start with a tiny, consistent dataset.

Use roughly 10 to 20 images for the first pass. Keep the style tight and avoid mixing wildly different aesthetics together.

Step 02

Pick the right focus before you pick strength.

Choose Stylized Look for kids-drawing style work. Choose Quick Test for the first run even if you already know you want the style to hit hard later.

Step 03

Fill in every caption before you hit start.

Use Open Pictures Folder, then Open Captions File, and make sure every image has a real caption. The current flow intentionally stops you from training on blank text.

Step 04

Treat Quick Test as a learning run.

If it completes and the result bends output toward your style, that is a win. You can move to Normal next. Save Strong for when you already know the dataset is healthy and simply needs more push.

// Results

How to read what comes out.

The first LoRA does not need to be perfect to be useful. What matters is whether it is pointing in the right direction and whether the failure mode tells us what to change next.

// Healthy Signals

What success looks like.

A healthy first run finishes cleanly, writes a LoRA file, and clearly nudges output toward the intended look. Even if it only feels halfway there, that is enough information to improve the next run.

// Trouble Signals

What failure usually means.

If the result is weak, the style signal may need more data or more training. If it becomes mushy, strange, or too aggressive, the dataset may be messy or the run may be pushing too hard for the data quality.

If It Is Weak

Try a cleaner or slightly longer second pass.

Move from Quick Test to Normal, tighten captions, and make sure the images are all really teaching the same visual language.

If It Is Overcooked

Do not just crank it harder.

Look for noisy captions, mixed styles, or images that do not belong in the set. A stronger run on a confused dataset usually gives a stronger mess.
