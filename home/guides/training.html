Guide 05

Z-Image TurboAI Toolkit LoRA Training Guide

Choose settings Add and start the job Advanced AI Toolkit

This guide starts at the trainer screen because that is where the practical setup happens: name the LoRA, open the pictures folder, put the images in place, review captions if you want them, choose the settings, click Add Job, then click Start Job. The Manager is just the front end. AI Toolkit is the real job runner underneath.

  • Good images and a tight visual target matter more than any single magic setting.
  • Open Pictures Folder is part of dataset prep and it lives right here in the trainer screen.
  • Caption with Brain is optional and lives inside the same setup stage.
  • Add Job happens after you decide the settings, not before.
  • Start Job only runs the saved AI Toolkit job.
  • The first run is about getting signal, not perfection.

The shortest honest version is this: put the right images in first, keep the dataset coherent, use captions as support instead of as the main event, and judge the run by its samples and checkpoints rather than by theory.

  • Use Open Pictures Folder here to build or update the dataset.
  • Use captions to describe what is in each image only when you need them.
  • Use Fast Test for quick signal and Baseline for the safest normal first run.
  • Use the advanced section at the end only if you want to stay inside the upstream AI Toolkit UI.

1. Set up the training job.

This is the part that comes before job creation. In the Manager, this is also where you prepare the dataset with Open Pictures Folder, then choose the preset, adapter, sample prompt, steps, checkpoints, learning rate, rank, and whether to use Low VRAM mode. This section explains the NymphsCore Manager screen shown below.

LoRA name: this is the training/job name. Keep it short and specific so you can recognize the output files later. If you use a trigger word, it is often simplest to keep the name close to that trigger so you do not forget what to prompt.
Open Pictures Folder: this is where dataset prep happens. It opens the image folder for the current LoRA name so you can add, remove, or replace the actual training images before you start worrying about the rest of the settings.
Open LoRAs: opens the output folder where finished LoRAs and checkpoints end up.
Caption with Brain: this is an optional helper inside the setup stage, not a separate major step. It drafts one editable caption per image into metadata.csv.
Caption fill: this is not a training quality knob. It only controls how Caption with Brain writes draft caption rows. Fill Blank Only keeps anything you already wrote. Overwrite All Drafts regenerates everything.
Open Draft CSV: this opens the editable caption sheet. When you later click Add Job, the app mirrors those captions into the per-image .txt files that AI Toolkit actually trains from.
How to caption style work: describe the content normally instead of padding every line with repeated style labels. The images should teach the rendering style. The captions should teach what to ask for.
Preset choice: Fast Test is the quickest path when you just want to see whether the LoRA is learning at all. Baseline is the safest normal first run. Style is the normal style preset. Strong Style is the follow-up when the normal style run still feels too polite.
What the presets currently write in Manager: Fast Test sets character mode, quick training, 500 steps, checkpoints off, 1e-4 learning rate, rank 8. Baseline sets character mode, normal training, 3000 steps, 4 checkpoints, 1e-4 learning rate, rank 16. Style uses the same numbers as Baseline but switches to style mode. Strong Style pushes that to 5000 steps with 4 checkpoints and rank 16.
Adapter choice: stay on v1 (Recommended) for the first serious run. Use v2 (Experimental) only after the normal path is behaving and you actually want a comparison.
Sample prompt: this matters more than it looks. AI Toolkit uses it to generate preview samples while the job is running, so it should reflect the kind of image you actually want to judge during training.
Steps: this is the total training length. More steps give the LoRA more chances to lock onto the concept, but more is not automatically better. For Z-Image Turbo, a practical starting point is usually 3000 for a normal run, with 500 used only for a quick signal check.
Checkpoints: this controls how many intermediate save points the Manager keeps available for you to test. Off means you only care about the final output. 2, 4, or 8 gives you fallback versions in case the final result is too weak or too cooked.
Learning rate: this is how hard each step pushes the LoRA. Too low and training crawls. Too high and it can overlearn or destabilize. A solid starting point for Z-Image Turbo is still 1e-4. The Manager also exposes 8e-5 and 5e-5 if you want a gentler run.
LoRA rank: rank is part of the LoRA's capacity. Lower ranks are lighter and can be enough for quick tests or simpler concepts. Higher ranks can hold more detail, but they are not automatically better. In this trainer, rank 8 is the quick-test setting and rank 16 is the normal default.
Low VRAM mode: turn this on when your GPU is the bottleneck and the normal path does not fit comfortably. The tradeoff is speed. It exists to get the run through on smaller cards, not because it improves quality by itself.
Add Job, Start Job, Stop Job, Delete Job: these are not interchangeable. Add Job writes the current settings into the real AI Toolkit job. Start Job runs that saved job. Stop Job stops the active run. Delete Job removes the saved job entry, so use that one deliberately.
Simple rule: choose the smallest set of settings that answers the question you have. If you only want to know whether the dataset works at all, use the quick path. If you want the safest normal proof, use Baseline. Save heavier style pushes for later.
Job name and trigger word: the training name is just the LoRA/job name. A trigger word is optional. If you use one, keeping the job name and trigger word close to each other makes it easier not to forget what to prompt later.
Steps, save every, and checkpoints: steps are the total length of the run. Save every controls how often a checkpoint is written. Max steps to keep saved controls how many of those checkpoints survive instead of being overwritten. In plain English: longer training and more saved checkpoints give you more chances to find the sweet spot, but they also cost more time and disk space.
Batch size and gradient accumulation: batch size is how many images the job sees at once. Bigger batches usually give the model a better sense of the concept, but they use more VRAM. Gradient accumulation is the slower workaround when you want batch-like behavior without actually fitting that bigger batch in memory.
Optimizer, learning rate, and weight decay: AdamW8Bit is a strong normal default when VRAM allows it. Learning rate is how hard each step pushes the LoRA. Too low and it barely learns. Too high and it overlearns or breaks down. Weight decay is a light anti-overcooking control that helps stop the model from clinging too hard to exact training images.
Timestep type and bias: these control where training pays attention across the denoising process. Balanced is the normal safe start. High noise is more about earlier composition-building stages and can help when you are trying to force a much stronger overall style shift. For style runs, it usually makes sense to start balanced and only lean harder into high noise if the style still is not taking over enough.
Text encoder and unload text encoder: the text encoder is what helps the model understand prompt words more specifically. Turning it off or unloading it is mostly a VRAM tradeoff. If you are memory-constrained, unloading it can help, but it slows training down because it keeps having to reload that component.

2. Click Add Job, then Start Job.

These are two separate steps. Add Job is the commit point. Start Job is the run point. The app is designed around that distinction.

Add Job: prepares the dataset support files, refreshes metadata.csv, mirrors the captions into per-image .txt files, writes the AI Toolkit YAML job, writes the LoRA metadata, and registers the job with the official AI Toolkit UI.
Start Job: queues and starts the saved AI Toolkit job. If you changed settings after the last Add Job, the app will stop you and tell you to apply those pending changes first.
Real Flow
1. Pick the dataset / LoRA name 2. Open Pictures Folder and add the images 3. Optionally use Caption with Brain 4. Review metadata.csv if needed 5. Choose preset, adapter, sample prompt, and training settings 6. Click Add Job 7. Click Start Job

3. Use samples and checkpoints to judge the run.

The first run does not need to be perfect to be useful. What matters is whether the samples and checkpoints show the LoRA moving in the right direction. That is the signal you use to decide whether to keep, adjust, or rerun.

What success looks like: the job finishes cleanly, writes checkpoints or a final LoRA, and the training samples visibly move toward the intended character, style, or concept. Halfway there is still useful information.
What weak results usually mean: the dataset may need to be cleaner, the style signal may need a stronger preset or longer run, or the sample prompt may not be showing the thing you actually care about.
What overcooked results usually mean: do not just crank it harder. Look for noisy captions, mixed images, or a dataset that is trying to teach too many different things at once.
Checkpoint testing: start with the later checkpoint or final file first, then compare backward if it feels too weak or too aggressive.

4. Test the LoRA in Blender.

Once the run is done, the Blender side should be simple: enable the LoRA, pick the checkpoint or final file, prompt normally, and only start comparing variations after you have seen one clean first result.

First test: choose the LoRA in the Z-Image panel, leave the recommended activation on for the first run, and generate once before changing several things at the same time.
Prompting: prompt the scene or subject normally. The activation phrase is only there to help some LoRAs lean into their learned look more clearly if they need that extra nudge.
Comparison order: if the result is weak, keep activation on first, then adjust LoRA strength, then compare checkpoints. If the result is already strong, you can try turning activation off to see whether it is still needed.

5. Advanced: if you want to do the whole run in the AI Toolkit UI.

This section is for advanced users who want to work directly in the upstream AI Toolkit UI instead of staying in the simplified Manager layer.

GPU ID: only matters if the machine has more than one GPU. It tells AI Toolkit which card should do the training job.
Trigger word: if you set one here, AI Toolkit can prepend it to the training text. Use it when you want a deliberate activation token. Skip it when you want the LoRA to behave more naturally from normal prompting.
Starting point: for the AI Toolkit UI, a common starting direction is AdamW8Bit, 1e-4 learning rate, weight_decay 1e-4, weighted timesteps, MSE loss, EMA off, cache text embeddings on, and a balanced bias before moving into a stronger style push later.
Save dtype: this is the precision format used when the checkpoints or LoRA files are written. BF16 is a good practical default because it is stable and light enough for most modern training runs.
Save every, sample every, and sample steps: save every controls how often AI Toolkit writes a checkpoint. sample every controls how often it renders preview images. It is usually smartest to keep those aligned so each saved checkpoint has matching preview samples. sample steps is just the number of denoising steps used to render those previews.
Batch size and gradient accumulation: batch size is how many images the job sees at once. Bigger batches usually give the model a better sense of the concept, but they use more VRAM. Gradient accumulation is the slower workaround when you want batch-like behavior without actually fitting that bigger batch in memory.
Optimizer, learning rate, and weight decay: AdamW8Bit is the normal default when VRAM allows it. Learning rate is how hard each step pushes the LoRA. Too low and it barely learns. Too high and it overlearns or breaks down. Weight decay is a light anti-overcooking control that helps stop the model from clinging too hard to exact training images.
Loss, timestep type, timestep bias, and noise type: MSE is the normal loss choice here. Timestep settings control where training pays attention across the denoising process. Balanced is the safe start. High noise shifts attention earlier in the denoising process, which can help with stronger style takeover or bigger composition changes. Newer models use flow match while older SD-style models use DDPM.
EMA and EMA decay: EMA is a smoothing option. Leaving it off gives a more direct baseline. Turning it on smooths the update path, which can make results more consistent on some smaller or fussier runs. EMA decay is just the strength of that smoothing.
Text encoder, cache text embeddings, and unload text encoder: the text encoder helps the model tie prompt words to the concept more specifically. Caching text embeddings is the convenient speed-and-VRAM choice when the captions are static. Unloading the text encoder saves VRAM, but it slows training because AI Toolkit has to keep reloading that component.
What those settings mean together: the baseline logic is simple. Start balanced, keep the learning rate sane, use an optimizer that fits your VRAM, save checkpoints at useful intervals, and only lean into more aggressive noise bias or experimental tweaks when the normal run is clearly too weak.
Style-specific note: for extreme style changes, start from a balanced setup and only later switch toward a high noise bias if you need the model to rebuild more of the early composition. That is useful later, but it is not the first thing to reach for on every run.
Trigger words and captions: trigger words are optional. Some runs use them, some do not. For style work, it usually makes more sense to caption the content normally and let the images teach the style.
Direct AI Toolkit UI Order
1. Datasets -> New Dataset 2. Add the images 3. Add captions if you want them 4. New Job 5. Set training name and trigger word if using one 6. Select Z-Image Turbo and the training adapter 7. Set prompts and training settings 8. Create Job 9. Training Queue -> Start 10. Judge samples and checkpoints
