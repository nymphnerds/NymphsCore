Guide 01

Install and first launch.Know what “ready” looks like.

This guide walks through the first healthy setup path for NymphsCore. The goal is simple: install the manager, let it finish its prep work, and make sure the runtime looks ready before you move into the creative workflow.

Read the steps Watch the demo slot Back to guide hub
// Walkthrough

From download to healthy first launch.

You are looking for a stable ready state, not just an installer that opened. This page focuses on the signals that tell you the setup completed cleanly.

Step 01

Run the manager installer

Start with the manager and let it complete its preparation work. The first run may take longer than later runs because it is preparing the local runtime environment.

Step 02

Wait for setup, not just window launch

A launched app window is not the same thing as a fully prepared backend. Let the setup process finish before judging whether the install worked.

  • Watch the status text for real progress.
  • Expect model or environment preparation to take time.
  • Do not jump straight into Blender if setup still looks active.
Step 03

Check for a healthy ready state

Before moving on, confirm that the manager no longer looks stuck and that the runtime tools or health checks report a usable state.

Step 04

Move into the addon only after the backend is sane

If the setup step already feels unclear or unstable, stop there and report it. It is much easier to fix first-run issues before they get mixed into creative workflow problems later.

// What To Expect

A good first run should feel clear.

You should be able to tell whether setup is still working, whether it finished, and whether the next move is Blender or runtime checks. If that story is muddy, that is worth reporting.

// If It Feels Wrong

Stop at the first confusing point.

Do not wait for a full crash to count it as a problem. Long silent waits, misleading status, or unclear next steps are all useful bug report material.
