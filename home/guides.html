// Start Here

What belongs on GitHub Pages.

GitHub Pages is a good fit for guide pages, screenshots, lightweight JavaScript, and embedded YouTube videos. It is not a good fit for hosting lots of large raw video files directly in the repo.

Recommended content mix

Static guides + external video hosting.

  • Write the guide pages here in plain HTML for now.
  • Keep screenshots and UI images in the repo.
  • Upload short demos to YouTube when ready.
  • Embed each demo with an iframe later.
  • Use the bug report page as the support endpoint from every guide.
Why this works

Low friction to publish.

You avoid bloating the repo with large video files, keep the site fast, and still get clean demo embeds inside the guide flow.

When you have a YouTube URL later, each placeholder block below can be swapped for a real embed using the `youtube.com/embed/VIDEO_ID` format.
