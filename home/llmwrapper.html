Cached LLM Wrapper.
Smart cloud delegation without losing local flow.

Keep everyday work on your local model, then hand off the expensive reasoning jobs to stronger cloud models only when needed. The wrapper adds prompt caching, one-shot tool calls, and a clean MCP path back into your local Brain stack through Manager or shell setup.

// Why it exists
Use local by default.
Escalate only when it matters.

This wrapper keeps the Brain stack grounded in local inference while still letting you reach GPT, Claude, Gemini, DeepSeek, and free OpenRouter options for the hard stuff when you choose to enable it.

  • Identical prompts return instantly from local cache.
  • llm_call stays one-shot to avoid recursive loops.
  • Works cleanly with the MCP bridge and Open WebUI tool routes.

// Core behavior

What it is.
What it adds.

The wrapper is not meant to replace your local Brain model. It acts more like a selective cloud gateway with memory for repeated prompts.

Cloud AI Gateway

Your local 27B model handles normal tasks. When something needs more reasoning power, the wrapper calls stronger hosted models through OpenRouter.

Prompt Caching

Identical prompts are hashed and cached locally, so repeated calls come back instantly and burn fewer credits.

One-Shot by Design

llm_call returns a single framed answer instead of opening a looping conversation between models.

The wrapper stays intentionally narrow: one tool for the cloud query, plus utilities for cache visibility and cleanup.

// Config

Files created.
Defaults worth knowing.

Installation drops a few core files into the Brain stack and gives you a small set of cache-related defaults that can be tuned later. If no OpenRouter key is present, the Brain stack simply skips the wrapper instead of blocking startup.

Nymphs-Brain/secrets/llm-wrapper.env Stores your OpenRouter API key.
Nymphs-Brain/bin/mcp-start MCP proxy config with llm-wrapper settings.
Nymphs-Brain/mcp/data/llm_cache/prompt_cache.sqlite SQLite cache database for stored prompt results.
Nymphs-Brain/local-tools/remote_llm_mcp/cached_llm_mcp_server.py Bundled cached LLM MCP runtime written by the Brain installer.
Nymphs-Brain/mcp/config/open-webui-tool-servers.json Brain-managed Open WebUI tool routes, including llm-wrapper when enabled.

// Install

Enable it.
Manager first, shell if you want.

The easiest path is now inside NymphsCore Manager. Enter an OpenRouter key on the Brain page, apply it, and start or refresh Brain. The shell installer still exists for direct manual installs and advanced tweaking.

Manager flow: Open the Brain page, enter your OpenRouter key on the one-line OpenRouter key row, click Apply Key, then use Manage Models to choose the remote OpenRouter model for llm-wrapper. After that, start Brain or run Update Stack. When the key exists, Brain adds llm-wrapper to MCP, mcpo, and Open WebUI automatically. When the key is missing, Brain leaves the wrapper out and the rest of the stack still starts normally.
Direct test: Once Brain is running, you can verify the wrapper itself with curl -s http://127.0.0.1:8099/llm-wrapper/llm_call -H 'Content-Type: application/json' -d '{"prompt":"Reply with exactly DIRECT_WRAPPER_TEST_OK and nothing else."}'. Run it twice and the second response should usually come back as a cache HIT.
# Manual shell flow # Step 1: Make the script executable cd /path/to/remote_llm_mcp chmod +x install-llm-wrapper.sh # Step 2: Optional preview ./install-llm-wrapper.sh --dry-run # Step 3: Run the installer ./install-llm-wrapper.sh
WSL support: The shell installer automatically detects WSL and adjusts the MCP proxy binding so Windows applications can connect correctly through localhost forwarding.
// Runtime contents

What Brain installs.
What each file is for.

From the Manager path, Brain writes the cached runtime into its own install tree and wires the rest of the stack around it. The original shell bundle still exists as a manual fallback, but it is no longer the main setup story.

cached_llm_mcp_server.py The bundled MCP runtime that provides llm_call, cache_stats, and cache_clear.
llm-wrapper.env Stores the OpenRouter key that turns the wrapper on.
open-webui-tool-servers.json Seed file that makes the Brain wrapper show up in Open WebUI tools when enabled.
// One-shot mode

Prevent multi-turn loops.
Return one answer cleanly.

Without one-shot framing, the local LLM can treat the remote answer like the next turn of a conversation and keep calling back out. One-shot mode makes the boundary explicit.

graph LR A[=== REMOTE LLM RESPONSE ===] --> B[2+2 equals 4.] B --> C[=== END REMOTE LLM RESPONSE ===] C --> D{Local AI} D -->|Process answer only| E[Respond to user
Do NOT call llm_call again]
