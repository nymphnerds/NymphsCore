Nymphs Top Down Camera (Nymphs TDC) - Comprehensive User Guide

### 1. Welcome to Nymphs TDC
Nymphs TDC is a professional, modular top-down camera and player controller system. It is designed to be "character-agnostic," meaning it works with any character model, whether it's a simple primitive or a complex, multi-bone character with a sophisticated Animator Controller.

### 2. Why Nymphs TDC is Universal
*   **Target-Based Camera**: The camera system does not rely on your character's internal hierarchy or bone structure. It simply follows the `Transform` (position) of the object you assign to it.
*   **Decoupled Architecture**: The `CameraController` and `PlayerController` are independent. You can use the camera without the controller, or use the `PlayerController` with a different camera system.
*   **Standard Unity Components**: It uses standard `Transform`, `CharacterController`, and `PlayerInput` components, ensuring compatibility with any project.

### 3. Step-by-Step Setup Guide

#### A. Setting up the Camera
1.  **Import**: Ensure the `Nymphs_TDC` folder is in your `Assets` directory.
2.  **Place the Camera**: Drag the `NTDC_Camera` prefab into your scene.
3.  **Assign Target**: Select the `NTDC_Camera` in the Hierarchy. In the `CameraController` component, drag your character GameObject into the **Target** field.
4.  **Clean Up**: Disable or delete the default `Main Camera` that came with your scene to avoid conflicts.
5.  **Customize**: Adjust the `Distance`, `Pitch`, and `Zoom` settings in the `CameraController` inspector to get the exact top-down feel you want.
6.  **Input Actions**: Ensure your `PlayerInput` uses the provided `GameInput` actions asset so `Look` and `Zoom` work (mouse wheel + gamepad right stick Y by default).

#### B. Setting up the Player Controller
1.  **Add Component**: Select your character GameObject and add the `PlayerController` script.
2.  **Dependencies**: Ensure your character has a `CharacterController` and `PlayerInput` component attached.
3.  **Configure States**: In the `PlayerController` component, you will see fields for `Idle State Name`, `Walk State Name`, and `Run State Name`.
    *   Open your character's **Animator Controller**.
    *   Find the exact names of your Idle, Walk, and Run states.
    *   Type these exact names into the corresponding fields in the `PlayerController` inspector.

### 4. How to handle Sprinting (The "Easy" vs "Pro" Way)

#### The "Easy" Way (Default)
By default, the `PlayerController` uses **Animation Speed Scaling**. When you hold Shift or Gamepad B, the controller increases the movement speed AND the playback speed of your animation. This makes your character look like they are running without needing any extra setup in the Animator.

#### The "Pro" Way (Custom Run Animation)
If you have a dedicated "Run" animation and want to use it instead of speeding up the walk:
1.  **Open Animator**: Double-click your character's Animator Controller.
2.  **Create State**: Right-click in the Animator window and select **Create State** -> **Empty**.
3.  **Name it**: Rename the new state to "Run" (or your preferred name).
4.  **Assign Clip**: Drag your run animation clip into the **Motion** field of this new state.
5.  **Update Script**: Go back to your character in the Hierarchy and update the `Run State Name` field in the `PlayerController` component to match the name you just gave the state (e.g., "Run").
6.  **Test**: Press Play. The controller will now automatically switch to this state when you hold Shift or the Gamepad B button.

### 5. Advanced: Using Blend Trees
For professional-grade, smooth transitions between walking and running:
1.  In your Animator, create a new **Blend Tree** state.
2.  Set the parameter to "Speed".
3.  Add your Walk and Run animations to the Blend Tree.
4.  The `PlayerController` will automatically handle the blending if you set up your Animator correctly.

### 6. Detailed Settings Reference

#### A. CameraController Settings
Target Settings
*   **Target**: The transform the camera follows. If empty, Auto Find Target can assign one at runtime.
*   **Auto Find Target**: If enabled, the camera searches for a target when none is assigned.
*   **Target Tag**: Tag used when auto-finding a target. Default is `Player`.
*   **Target Search Interval**: Seconds between auto-find attempts.
*   **Height Offset**: Vertical offset added to the target position.

Input Settings
*   **Player Input**: The PlayerInput component to read actions from. If empty, the camera tries to find one.
*   **Look Action Name**: Action name used for camera rotation (default `Look`).
*   **Zoom Action Name**: Action name used for zoom (default `Zoom`).
*   **Use Legacy Input Fallback**: If true, uses Mouse/Gamepad direct reads when no actions are found.

Zoom Settings
*   **Distance**: Current zoom distance (camera to target).
*   **Min Distance**: Closest allowed zoom.
*   **Max Distance**: Farthest allowed zoom.
*   **Zoom Speed**: Base zoom rate.
*   **Zoom Smooth Time**: Smoothing time in seconds. `0` = instant. Higher = longer tail.
*   **Mouse Wheel Zoom Multiplier**: Multiplier for mouse wheel zoom steps. Lower = smaller steps.
*   **Gamepad Zoom Dead Zone**: Minimum right stick Y before zoom engages.
*   **Gamepad Zoom Strength**: Multiplier for gamepad zoom rate.

Rotation Settings
*   **Mouse Rotation Speed**: Horizontal rotation rate for mouse drag.
*   **Gamepad Rotation Speed**: Horizontal rotation rate for right stick X.
*   **Gamepad Rotation Dead Zone**: Minimum right stick X before rotation engages.
*   **Mouse Look Action Sensitivity**: Extra multiplier for mouse Look action to match raw mouse delta feel.

Dynamic Pitch Settings
*   **Min Pitch**: Pitch angle at minimum zoom.
*   **Max Pitch**: Pitch angle at maximum zoom.

Tuning Tips
*   **Snappy zoom**: Set `Zoom Smooth Time` to `0.0` to remove all tail. Use `0.05` to `0.1` for a quick, gentle stop.
*   **Smaller mouse wheel steps**: Lower `Mouse Wheel Zoom Multiplier` and/or `Zoom Speed`.

#### B. PlayerController Settings
Movement Settings
*   **Move Speed**: Base movement speed in units per second.
*   **Camera Transform**: The camera used for camera-relative movement. If empty, uses `Camera.main`.
*   **Sprint Multiplier**: Multiplies Move Speed when sprinting.
*   **Rotation Speed**: How fast the character turns toward the movement direction.
*   **Gravity**: Downward force when airborne.

Input Smoothing
*   **Input Smooth Time**: How long it takes to blend to a new input direction. Lower = snappier. Higher = smoother.

Animation Settings
*   **Idle State Name**: Animator state name for idle.
*   **Walk State Name**: Animator state name for walking.
*   **Run State Name**: Animator state name for sprinting.
*   **Animation Crossfade**: Time to blend between animation states.
*   **Base Anim Speed**: Base animation playback speed.

Tuning Tips
*   **Snappier WASD**: Lower `Input Smooth Time` (try `0.05` or `0.0`).
*   **Slower turning**: Lower `Rotation Speed`.

### 7. Mouse Follow + NavMesh Notes (Beginner Friendly)
Mouse Follow (Current Behavior)
*   Uses a **raycast** from the camera through your cursor to find a point on the ground.
*   Your ground **must have a collider** (mesh collider or box collider).
*   `Mouse Follow Ground Mask` lets you restrict clicks to the ground layer only.
*   **Hold LMB**: player follows the cursor continuously while held.
*   **Click once**: player remembers that point and walks there.
*   **Shift + click** while already moving: queues additional destinations in order.
*   **Click Indicator Prefab**: optional visual flash spawned at the click point.
*   If no prefab is assigned, a simple built-in **ring** is used (enabled by default).
*   Ring tuning: `Runtime Click Indicator Segments` (3-256; 4 gives a square/diamond outline) and `Runtime Click Indicator Line Width` (thickness). These are ignored if an indicator prefab is assigned.
*   `Click Indicator Pulse Until Arrived` keeps the indicator alive until the player reaches the destination.
*   `Click Indicator Pulse Speed` works whether Pulse Until Arrived is on or off.
*   `Click Indicator Pulse Repeat Count` limits pulse cycles (0 = infinite).
*   The player moves **directly** toward the point, so it will not path around obstacles.

NavMesh (Correct RPG Pathfinding)
*   A **NavMesh** lets the character find a path around walls/props.
*   The proper workflow is: Raycast to get a point, then call `NavMeshAgent.SetDestination(point)`.
*   Enable `Use NavMesh For Mouse Follow` and add a `NavMeshAgent` to the player.
*   The controller will use NavMeshAgent for click-to-move when LMB is held, and fall back to direct movement if no NavMesh is available.
