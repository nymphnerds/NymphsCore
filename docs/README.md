# NymphsCore Docs

Start here:

- [Nymphs Module Making Guide](NYMPHS_MODULE_MAKING_GUIDE.md) is the
  source-of-truth community module standard.
- [Nymph Module UI Standard](NYMPH_MODULE_UI_STANDARD.md) defines the boundary
  between the Manager shell and installed module-owned UI.

Supporting planning notes:

- [Current Nymph Module Repo Deep Dive](Ideas/CURRENT_NYMPH_MODULE_REPO_DEEP_DIVE.md)
  maps existing and planned modules into the `nymph.json` plus registry shape.
- [Nymph Plugin Standardization Handoff](Ideas/NYMPH_PLUGIN_STANDARDIZATION_HANDOFF.md)
  records the current proof state and recent session notes.
- [Nymph Manifest Draft](Ideas/NYMPH_MANIFEST_DRAFT.md) is historical manifest
  design context. Prefer the Module Making Guide when there is a conflict.

Standardization should happen by testing real modules one at a time, then
updating the Module Making Guide with whatever the proof module teaches.
